# Getting Started

SandboxFusion is a secure, multi-language code execution sandbox for running and evaluating code generated by LLMs. It exposes a simple HTTP API that accepts source code, runs it inside an isolated environment, and returns stdout/stderr/return-code.

## Capabilities

1. **Code Runner** -- Execute arbitrary code snippets in 28 CPU languages and return stdout, stderr, and the process exit code.
2. **Evaluation** -- Submit an LLM-generated completion together with inline stdin/stdout test cases. The sandbox extracts code from the completion, runs each test case, and reports pass/fail.

## Supported Languages

SandboxFusion supports the following languages and execution modes. The `language` value in parentheses is the identifier you pass to the API.

| C# | `csharp` | .NET SDK 10.0 |
| Go | `go`, `go_test` | Go 1.25.9; `go_test` runs via `go test` |
| Java | `java`, `junit` | JDK 25; `junit` runs via JUnit 5 |

## Installation

### Option 1: Docker (recommended)

Docker mode builds two images: a **base image** with all 20+ language runtimes pre-installed, and a **server image** that layers in the SandboxFusion application.

**Prerequisites:** Docker installed and running.

```bash
# Clone the repository
git clone https://github.com/<your-org>/SandboxFusion.git
cd SandboxFusion

# Build the base image (installs all language runtimes -- takes ~20 min first time)
make build-base-image       # produces ineil77/sandbox-fusion-base:24042026-4

# Build the server image (adds the app + Python deps on top of ineil77/sandbox-fusion-base:24042026-4)
make build-server-image     # produces ineil77/sandbox-fusion-server:24042026-4

# Run the server
docker run -d --rm --privileged -p 8080:8080 ineil77/sandbox-fusion-server:24042026-4
```

The `--privileged` flag is required because the sandbox uses overlayfs, cgroups, and network namespaces for lite isolation. If you run with `full` isolation mode, also mount the Docker socket and share a temp directory:

```bash
docker run -d --rm --privileged -p 8080:8080 \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -v /tmp/sandbox-shared:/tmp/sandbox-shared \
    -e SANDBOX_CONFIG=full_test \
    -e SANDBOX_TMP_DIR=/tmp/sandbox-shared \
    ineil77/sandbox-fusion-server:24042026-4
```

Nested Docker is **not** used; execution containers are siblings on the host daemon. See the [Isolation Modes](isolation-modes.md) document for details.

The server is now available at `http://localhost:8080`.

### Option 2: Manual Installation (Linux only, for lite isolation)

Manual installation installs all language runtimes directly onto the host OS. This is suitable for development or bare-metal deployment on Linux.

**Prerequisites:**
- Linux (required for overlayfs, cgroups, network namespaces)
- [Miniconda](https://conda.io/projects/conda/en/latest/user-guide/install/index.html) or [Conda](https://docs.conda.io/en/latest/)
- [Poetry](https://python-poetry.org/docs/#installation)
- The language runtimes listed in `scripts/Dockerfile.base` (Go 1.25.9, GCC 15.2, JDK 25, Node.js 24.0.0, .NET 10.0, Rust 1.95.0, PHP 8.5, Bash 5.3.9, R 4.5.2, Lua 5.2, Ruby 4.0.0, Julia 1.11.5, Scala 3.8.3, Perl 5.40.1, DMD 2.112.0, Kotlin 2.1.20, Icarus Verilog 13.0, Lean 4.29.0, Racket 9.1, Swift 6.1.2)

```bash
# Create and activate a Python 3.13 environment
conda create -n sandbox -y python=3.13
conda activate sandbox

# Install the Python application and its dependencies
poetry install

# Install all language-specific runtime dependencies
make install-runtimes
# This runs:
#   runtime/python  -- installs ~80 pip packages into a sandbox-runtime conda env
#   runtime/node    -- npm ci (installs Node.js deps including Jest, Babel, tsx)
#   runtime/go      -- go build (caches Go module dependencies)
#   runtime/lean    -- lake build (builds the Lean/Mathlib project)

# Create the docs build directory (required even if you don't build docs)
mkdir -p docs/build

# Start the server
make run          # development mode with hot-reload
# or
make run-online   # production mode (no reload)
```

The server listens on `http://0.0.0.0:8080` by default. Override with:

```bash
HOST=127.0.0.1 PORT=9090 make run
```

## Quick Verification

Health check:

```bash
curl http://localhost:8080/v1/ping
# Returns: "pong"
```

Run a Python snippet:

```bash
curl -X POST http://localhost:8080/run_code \
  -H "Content-Type: application/json" \
  -d '{"code": "print(sum(range(10)))", "language": "python", "run_timeout": 10}'
```

Expected response:

```json
{
  "status": "Success",
  "message": "",
  "compile_result": null,
  "run_result": {
    "status": "Finished",
    "execution_time": 0.016,
    "return_code": 0,
    "stdout": "45\n",
    "stderr": ""
  },
  "executor_pod_name": null,
  "files": {}
}
```

Run a C++ snippet (compiled language -- note `compile_result` is populated):

```bash
curl -X POST http://localhost:8080/run_code \
  -H "Content-Type: application/json" \
  -d '{
    "code": "#include <iostream>\nint main() { std::cout << \"Hello\" << std::endl; return 0; }",
    "language": "cpp"
  }'
```

Expected response:

```json
{
  "status": "Success",
  "message": "",
  "compile_result": {
    "status": "Finished",
    "execution_time": 0.458,
    "return_code": 0,
    "stdout": "",
    "stderr": ""
  },
  "run_result": {
    "status": "Finished",
    "execution_time": 0.002,
    "return_code": 0,
    "stdout": "Hello\n",
    "stderr": ""
  },
  "executor_pod_name": null,
  "files": {}
}
```

Submit an LLM completion for evaluation:

```bash
curl -X POST http://localhost:8080/submit \
  -H "Content-Type: application/json" \
  -d '{
    "id": "problem-1",
    "completion": "```python\na, b = map(int, input().split())\nprint(a + b)\n```",
    "config": {"language": "python"},
    "test_cases": [
      {"input": {"stdin": "1 2\n"}, "output": {"stdout": "3\n"}},
      {"input": {"stdin": "10 20\n"}, "output": {"stdout": "30\n"}}
    ]
  }'
```

## Running Tests

```bash
make test                        # All tests with 64 parallel workers (TEST_NP=64)
make test TEST_NP=8              # Override parallelism (recommended for full mode)
make test-case CASE=test_python  # Single test with stdout visible
make test-minor                  # Minor language tests only
```
